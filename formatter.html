Right now both the "Home Phone" and "Cell Phone" fields are using the formatter to
format user input. They are using different formats although both are commonly used
for phone numbers. I wanted to test formatter.js using both a format that started
with user input as the first character, and a format that started with a pattern as
the first character.

The "Get Format" button calls the function "formatter().getFormattedInput()". This function
takes in two parameters. The first is a format, and the second is the id of an input. The
function will take the current value in the field and will return the value reformatted in
the newly supplied format. It is currently used on the zip code field.

The "Remove Formatting" buttons calls the function "formatter().removeFormatting". This
function takes in a single parameter: an id of an input field. It will return a the input's
value without any formatting applied. It is currently used on the home phone field.

More will be added to this test page as I continue to fill out the functionality.
Feel free to mess around with the formats used in the inputs.
